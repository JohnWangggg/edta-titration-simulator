## TL;DR（一页纸结论）

1. **模拟器 Stage 2 的"缓冲液失效 → 越滴越粉"是真实化学现象，机制正确**——pH 跌破 6.3 后铬黑 T 游离态本身就呈红色，EDTA 把金属抢走也显示不出蓝色。这是终点不可见，不是"反应失败"。
2. **模拟器的 pH 公式是教学简化曲线，不是真实化学**——真实场景在无缓冲下滴定 25 mL × 0.05 mol/L 的 Zn²⁺，到 20 mL 时 pH 会跌到 ~1.3（模拟器钳在 3.5 是为了视觉可读）。教学目的下保留即可，但**文档里要说清楚这是简化**。
3. **pH 6.5 红线**：真实铬黑 T 变色点是 6.3，模拟器用 6.5 偏保守（更早报警）。可在 UI 里标注"真实阈值 6.3，模拟器为保守起见取 6.5"。
4. **🚨 Stage 2 化学有重大简化**：EDTA 不能直接滴定 SO₄²⁻（EDTA 只与金属阳离子络合）。真实方法是先用 BaCl₂ 沉淀 SO₄²⁻，再用 EDTA 反滴过量 Ba²⁺。模拟器把整个过程压缩成"EDTA 滴硫酸钠"，**不应作为真实实验流程教给朋友**。详见 §6。
5. **chrom 黑 T 颜色名称**：模拟器把"锁死"态标为 HOT_PINK（亮粉红），真实是"紫红 / 葡萄酒红"。教学夸张没问题，但要让学生知道"实际不是这么亮的粉色"。
6. **可扩展内容**：除了铬黑 T，还有钙指示剂、二甲酚橙、紫脲酸胺、PAN、甲基百里香酚蓝等；不同金属、不同 pH 用不同指示剂。可作为 Thread B 的素材。

## 1. 核心反应（EDTA 络合滴定的化学骨架）

**主反应**（滴定中实际发生的）：

```
M^n+ + H₂Y^(2-n)- ⇌ MY^(n-4)- + 2 H+
```

对二价金属（M²⁺，如 Zn²⁺、Mg²⁺、Ca²⁺）：

```
M²⁺ + H₂Y²⁻ ⇌ MY²⁻ + 2 H+
```

> **关键事实**：每络合一个 M²⁺，释放 2 个 H⁺。这就是为什么滴定过程中 pH 会持续下降——反应本身是产酸的。

**25 mL × 0.05 mol/L Zn²⁺ 的滴定**（完全反应时）：
- 总 Zn²⁺：1.25 mmol
- 总释放 H⁺：2.5 mmol
- 假设总体积到终点约 50 mL，理论最低 pH ≈ 1.30（强酸近似）

**条件稳定常数**（决定滴定能否定量进行）：
- lg K'(MY) = lg K(MY) − lg α_Y(H)
- 准确滴定判据：lg K'(MY) ≥ 8
- 对 Zn²⁺（lg K = 16.5），pH ≥ 4 即可；对 Mg²⁺（lg K = 8.7），需要 pH ≥ 9.7

## 2. 铬黑 T（EBT / Eriochrome Black T）指示剂

### 2.1 基本信息

- **化学名**：1-(1-羟基-2-萘偶氮基)-6-硝基-2-萘酚-4-磺酸钠
- **分子式**：C₂₀H₁₂N₃NaO₇S，分子量 461.38
- **CAS**：1787-61-7
- **最大吸收波长**：λmax = 612-616 nm（在 pH 10 缓冲液中）
- **物理形态**：黑褐色粉末，水溶液久置失效（数天）

### 2.2 颜色—pH 关系（游离态）

| pH 区间 | 颜色 | 说明 |
|---|---|---|
| < 6.3 | 紫红 / 酒红 | 偶氮基断裂形成醌式结构 |
| 6.3 ~ 11.6 | 蓝色 | 正常工作区 |
| > 11.6 | 橙色 | 酚羟基去质子化 |

**推荐使用范围**：pH 9 ~ 10.5（蓝区内偏下端）
**模拟器用的 6.5 vs 真实 6.3**：模拟器偏保守，更早锁定；教学上没问题但要在 UI 里标注。

### 2.3 颜色变化的化学原理（端点机制）

**滴定前**（有 M²⁺）：

```
M²⁺ + HIn²⁻(蓝) ⇌ MIn⁻(红) + H+
```

**游离 EBT 是蓝色，与 M²⁺ 络合后变成红色**——所以滴定前溶液是红的。

**滴定终点**（EDTA 把 M²⁺ 抢走）：

```
MIn⁻(红) + H₂Y²⁻ ⇌ MY²⁻ + HIn²⁻(蓝) + H+
```

EDTA 与 M²⁺ 的络合比 EBT 更强，终点时 M 从 EBT 上被"撬走"，游离出来的 EBT 是蓝色，溶液由红变蓝。

**🚨 关键（这就是模拟器演示的"粉色锁死"机制）**：

如果 pH < 6.3，**游离 EBT 本身就不是蓝色，而是紫红色**。所以：
- 终点时 MIn⁻（红）→ M-Y²⁻ + HIn²⁻（紫红，因为 pH 太低）
- 颜色没变化，**端点完全不可见**
- 操作者以为还没到终点，继续滴 → 过量

> **这正是模拟器"越滴越粉"的物理本质**——不是 EDTA 抢不到金属，是**指示剂在低 pH 下丧失显色能力**。这是一个非常优雅的、可以让学生记住一辈子的"底层变量崩溃导致上层表象失效"的例子。🧪

### 2.4 颜色优先级（在 `src/engine/physics.ts` 里要遵守的）

1. **`!bufferEnabled && pH < 6.5` → 强制 HOT_PINK**（粉色锁死）
2. `bufferEnabled` → 体积驱动（< 19.5 酒红 / 19.5-20.0 紫红 / 20.0-20.5 纯蓝 / ≥ 20.5 深蓝）
3. `!bufferEnabled && pH ≥ 6.5` → < 20.0 酒红 / ≥ 20.0 纯蓝（缓冲关闭但 pH 还没破线）

这条优先级是**模拟器的教学灵魂**，任何重构都不能破坏。

### 2.5 EBT 的两个常见失效模式（真实化学里的"指示剂事故"）

| 现象 | 原因 | 表现 |
|---|---|---|
| **封闭** (blockade) | MIn 太稳定（Fe³⁺、Al³⁺、Cu²⁺、Co²⁺、Ni²⁺），EDTA 抢不动 M | 溶液**始终红色**，加再多 EDTA 也不变色 |
| **僵化** (rigidization) | MIn 溶解度太小（生成胶体/沉淀），交换缓慢 | 终点**拖长**，颜色变化模糊 |

模拟器目前没演示这两个。但**真实实验里 Fe³⁺、Al³⁺ 微量污染**会直接导致 EBT 失效（"封闭"）。这是 Thread B / 学生错误的素材。

## 3. 氨-氯化铵缓冲液（为什么偏偏是它，pH 偏偏是 10）

### 3.1 缓冲液的作用

EDTA 络合反应持续释放 H⁺，如果没有缓冲：
- pH 持续下降（可达 pH ~1.3）
- α_Y(H) 增大 → 游离 Y⁴⁻ 浓度降低 → K'(MY) 减小 → 滴定突跃变小
- EBT 指示剂变色范围失效 → 终点不可见

### 3.2 为什么是氨-氯化铵（NH₃-NH₄Cl）

经典配制（多本教材一致）：
- 20 g NH₄Cl 溶于 500 mL 水
- 加 100 mL 浓氨水（ρ ≈ 0.9 g/mL）
- 稀释至 1000 mL
- 实际 pH ≈ 10.0~10.3

**为什么偏偏 NH₃-NH₄Cl？**
1. pH 在 EBT 推荐区（9-10.5）内
2. 氨本身是辅助络合剂——NH₃ 与 Zn²⁺、Cu²⁺、Cd²⁺ 形成氨合络离子（[Zn(NH₃)₄]²⁺ 等），**有助于防止金属离子水解沉淀**，同时不干扰 EBT 的终点显色
3. 廉价、易得、配制简单
4. 对 Ca²⁺、Mg²⁺ 体系（水硬度测定）效果最佳——这正是经典的"水硬度 EDTA 法"配方

### 3.3 真实场景：氨还会"消耗"一点 EDTA

在 Zn²⁺ 体系里，Zn²⁺ + 4NH₃ ⇌ [Zn(NH₃)₄]²⁺，Zn-NH₃ 络合物的稳定性（lg β ≈ 9.0）比 Zn-EDTA（lg K ≈ 16.5）弱得多，所以终点附近 EDTA 仍能抢走 Zn。但**确实会有微小的条件稳定常数损失**，这是真实实验中"用氨缓冲但 pH 不要过度"的折衷。

模拟器目前把缓冲液简化成"pH = 10.0 死锁"——这是合理的教学简化，无需精细建模。

## 5. 模拟器"pH 公式"的拆解与建议

```typescript
export function computePH(volume: number, bufferEnabled: boolean): number {
  if (bufferEnabled) return 10.0;
  if (volume <= 0) return 7.0;
  return Math.max(3.5, 7.0 - Math.log10(volume * 10 + 1) * 2.5);
}
```

**意图**：模拟无缓冲滴定过程中 pH 从 7 单调衰减到 ~3.5，形态像对数衰减曲线。

**问题**：
1. `volume*10 + 1` 是为了让 `volume = 0` 时不出 NaN；可以，但 +1 这一项让曲线起点被"压扁"了
2. `* 2.5` 是为了在 volume=20 处接近 pH 3.5——这个系数完全是教学拟合，没有化学依据
3. 真实化学（25 mL × 0.05 M Zn²⁺）应在 volume=20 时 pH ≈ 1.3-1.5

**建议（待作者决策）**：

- **A. 保持现状**：在文档/UI 里明确说明"这是教学曲线，非真实 pH"。最省事。
- **B. 改用更接近真实化学的公式**：例如基于"每滴定 1 mL EDTA 释放 0.1 mmol H⁺"逐步累积。代码更复杂，但更接近真实。
- **C. 用 pH 钳制改为 1.5 + 改公式**：保留视觉冲击，但更接近现实。

**默认建议**：选 A，在文档里加一句话说明就够了。**不要为了"化学正确"破坏教学效果**。

## 6. 🚨 重大发现：Stage 2 的"硫酸根直接滴定"不成立

### 6.1 真实情况

EDTA（乙二胺四乙酸）是**金属阳离子螯合剂**，不与阴离子（包括 SO₄²⁻、PO₄³⁻、CN⁻ 等）形成稳定络合物。

**国家标准 SL 85-1994**（水利行业）规定的硫酸根 EDTA 测定法：

1. 取水样 50 mL，加 1+1 盐酸酸化
2. 加过量 BaCl₂（0.0125 mol/L）——所有 SO₄²⁻ 沉淀成 BaSO₄
3. 加 MgCl₂-EDTA 混合液 + 氨-氯化铵缓冲液 + 铬黑 T
4. **用 EDTA 标准溶液滴定过量的 Ba²⁺**（+ 样品中原有 Ca²⁺/Mg²⁺）
5. **另取一份水样单独滴定 Ca²⁺+Mg²⁺ 总量**（空白扣除）
6. 计算：SO₄²⁻ 含量 = (V_total − V_CaMg − V_blank) × C_EDTA × 96.06

这是**间接滴定法（返滴定法）**，不是直接滴定。

### 6.2 模拟器当前的处理

模拟器把 Stage 2 描述为"测定硫酸钠样品中的硫酸根"，并当成直接滴定处理。这在**真实化学里是错的**。

### 6.3 对教学的影响

- **正面**：模拟器真正传递的教学价值是"缓冲液失效 → 越滴越粉"，这个机制对 Zn²⁺、Mg²⁺、Ba²⁺、Ca²⁺ 等**任何可用 EDTA 滴定的金属**都一样。换什么金属都不影响核心教训。
- **负面**：如果朋友学完这个去真实验室"用 EDTA 滴定硫酸钠"，会**完全不知所措**——因为 EDTA 根本不跟硫酸钠反应。

### 6.4 三个修法选项（待作者决策）

**选项 A**：保留"硫酸钠"标签，但在 onboarding / Stage 2 入口加一行说明
- "本关卡的'硫酸钠'是教学简化——真实实验用 EDTA 反滴定 Ba²⁺ 间接测 SO₄²⁻。重点关注缓冲液失效机制。"
- 改动最小，保留原汁原味

**选项 B**：把 Stage 2 改成"测定 ZnSO₄ 中 Zn²⁺"（更化学正确）
- 锥瓶里装的是 ZnSO₄ 溶液
- 滴定的是 Zn²⁺（EDTA 直接滴定，pH 10，铬黑 T）
- 与 Stage 1 标定的 Zn²⁺ 直接呼应——**数据继承逻辑天然成立**
- 改动稍大，但化学无瑕疵

**选项 C**：保持现状，不加说明
- 不推荐。朋友可能会误以为真的能直接滴硫酸根

**作者的选择**会决定后续是否要改 `需求.md` 和 UI 文案。我**强烈推荐 B**——既化学正确，又能强化"数据继承"这个训练目标。

## 8. 学生常见操作错误清单（Thread B 素材）

来源：分析化学实验教学文献与多个工业标准（如 GB/T 601-2002）。可直接转化为 Stage 3 关卡或 onboarding 提示。

### 8.1 仪器准备类

| 错误 | 后果 | 严重度 |
|---|---|---|
| 滴定管未用标准溶液润洗 | 标准溶液被稀释 → 浓度偏低 → 体积偏大 | 高 |
| 锥形瓶被润洗 | 样品被稀释 → 测定值偏低 | 高 |
| 滴定管下端有气泡 | 读数偏大 | 中 |
| 滴定管漏液（活塞松动 / 橡皮管老化） | 体积不准 | 中 |
| 读数时视线不水平（仰视/俯视） | 读数偏差 ±0.01-0.02 mL | 高 |

### 8.2 滴定操作类

| 错误 | 后果 | 严重度 |
|---|---|---|
| 摇瓶不规范（前后晃而非圆周运动） | 反应不完全 | 中 |
| 流速过快（>3 滴/秒） | 局部过量、终点拖后 | 高 |
| 变色即停（未等反应完全） | 终点偏早 | 中 |
| 终点后立即读数（未等管壁液体流下） | 读数偏大 | 中 |
| 近终点未改用"半滴"操作 | 过量 | 高 |
| 没用水冲洗瓶壁附着的溶液 | 部分样品未反应 | 中 |

### 8.3 化学判断类

| 错误 | 后果 | 严重度 |
|---|---|---|
| 忘了加缓冲液 | pH 崩溃 → 终点不可见 | **致命**（模拟器已建模） |
| 指示剂加太多 | 指示剂本身消耗滴定剂；变色迟钝 | 中 |
| pH 不在指示剂工作区间 | 变色不可见 | **致命** |
| Fe³⁺/Al³⁺/Cu²⁺ 污染（未加掩蔽剂） | EBT 封闭 → 终点无色变 | 高 |
| 滴定管用前未用被测液润洗 | 浓度被稀释 | 高 |

### 8.4 计算与判断类

| 错误 | 后果 | 严重度 |
|---|---|---|
| Stage 2 计算时用了理论浓度 0.05 而非标定浓度 | 最终结果错误 | **致命**（模拟器已建模） |
| 摩尔质量用错 | 数值错 | 中 |
| 滴定体积不在 20-40 mL 之间 | 相对读数误差大 | 中 |
| 平行测定极差 > 0.2% 仍取均值 | 数据不可靠 | 中 |

### 8.5 模拟器已建模的 vs 待建模

**已建模**（模拟器能演示的）：
- 缓冲液失效 → 粉色锁死
- Stage 2 计算时用错浓度（陷阱分支）
- Stage 1 滑块过快导致过量

**未建模**（可作 Thread B/C 扩展）：
- 滴定管未润洗（可表现为体积读数偏差）
- 锥形瓶误润洗
- Fe³⁺ 污染（指示剂封闭，永久红色不蓝）
- 近终点没改半滴（视觉提示 + 鼓励改用按钮）
- 读数视差（小操作即可触发）

## 9. 关于教学化处理（给作者的判断框架）

模拟器的核心训练目标是：
