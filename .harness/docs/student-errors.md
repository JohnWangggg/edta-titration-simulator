# 学生常见操作错误清单与教学落地（Thread B 成果）

> **作者**：楼楼大王（项目作者本人）
> **目的**：把真实实验里学生最容易犯的操作错误梳理清楚，并设计如何把它们织入模拟器
> **范围**：EDTA 络合滴定的全部操作环节（仪器准备、滴定操作、终点判断、计算）
> **完成时间**：2026-06-18
> **依赖**：`.harness/docs/chem-background.md`（化学原理）

---

## TL;DR

1. **学生错误大致分四类**：仪器准备类、滴定操作类、化学判断类、计算与判断类
2. **模拟器已建模 3 个最致命的**：缓冲液失效、用错数据源、过量滴定——这三个都是"整组数据报废"级的
3. **Thread B 的落地建议**：**A+B 方案**（轻量 onboarding 卡片 + 通关反思强化），**不引入 Stage 3**
4. **核心原则**：模拟器是教学工具不是"找茬游戏"，每个错误都要有"为什么错 + 错哪了 + 真实怎么避免"三件套，不能仅惩罚

## 1. 学生常见错误分类汇总

来源：分析化学实验教学文献 + GB/T 601-2002 + SL 85-1994 等国标 + 多个高校实验教学讲义。

### 1.1 仪器准备类（开始滴定前）

| # | 错误 | 化学后果 | 严重度 |
|---|---|---|---|
| 1 | 滴定管未用标准溶液润洗 | 标准液被管壁残留水稀释 → 消耗 V 偏大 → 测定结果偏高 | 高 |
| 2 | 锥形瓶用待测液润洗（**绝对禁止**） | 待测液"加倍"加到瓶里 → 消耗 V 偏大 → 测定结果偏高 | 致命 |
| 3 | 滴定管下端有气泡未排出 | 气泡占据部分滴定管路 → 滴定后气泡消失，V 读数虚高 | 高 |
| 4 | 滴定管漏液（活塞松动 / 橡皮管老化） | 部分 EDTA 漏出未参与反应，但被计入 V → 测定结果偏高 | 高 |
| 5 | 锥形瓶未洗净残留杂质 | 杂质可能消耗 EDTA 或金属离子 → 结果随机偏差 | 中 |
| 6 | 移液管未润洗直接取待测液 | 待测液被稀释 → 实际浓度低于计算值 → 测定结果偏低 | 高 |

### 1.2 滴定操作类（开始滴定后）

| # | 错误 | 化学后果 | 严重度 |
|---|---|---|---|
| 7 | 摇瓶不规范（前后晃而非圆周运动） | 反应不完全 → 局部 EDTA 过量 → 终点提前 / 拖后 | 中 |
| 8 | 流速过快（>3 滴/秒） | 局部 EDTA 过量、反应未达平衡 → 终点提前 / 拖长 | 高 |
| 9 | 变色即停（未等反应完全） | 终点偏早，V 偏小 | 中 |
| 10 | 终点后立即读数（未等管壁液体流下） | 内壁挂液滴未流下，V 读数偏大 | 中 |
| 11 | 近终点未改用"半滴"操作 | 一滴 0.05 mL 直接过量 → 测定结果明显偏高 | 高 |
| 12 | 半滴操作不规范（直接滴到瓶里） | 局部过量、终点点过 | 中 |
| 13 | 锥瓶内溶液溅到瓶壁上方未冲下 | 部分样品未反应 → V 偏小 | 中 |
| 14 | 滴定管尖嘴外挂一滴标准液未滴落 | V 读数虚增 | 中 |

### 1.3 化学判断类（与化学原理直接相关）

| # | 错误 | 化学后果 | 严重度 |
|---|---|---|---|
| 15 | **忘了加缓冲液** | pH 跌破 6.3 → 铬黑 T 游离态变红 → **终点不可见** | **致命** ✅ 模拟器已建模 |
| 16 | 指示剂加太多（>5 滴） | 指示剂本身消耗金属离子 + 变色迟钝 | 中 |
| 17 | 用了错误的指示剂（如在 pH 12 用铬黑 T） | 指示剂变色范围失效 → 颜色突变点偏离终点 | 高 |
| 18 | **Fe³⁺ / Al³⁺ 微量污染未掩蔽** | 这些离子与 EBT 形成极稳定络合物（"封闭"现象） → 加再多 EDTA 也变不到蓝色 | **致命** |
| 19 | Cu²⁺ / Co²⁺ / Ni²⁺ 未掩蔽 | 同上，封闭 EBT | 致命 |
| 20 | pH 调到指示剂工作范围外 | 终点颜色变化不明显 | 致命 |

### 1.4 计算与判断类

| # | 错误 | 化学后果 | 严重度 |
|---|---|---|---|
| 21 | **用理论浓度 0.05 而非标定浓度** | 数据源错误 → 所有下游结果全错 | **致命** ✅ 模拟器已建模 |
| 22 | 摩尔质量用错（Na vs Zn，SO₄ vs SO₃） | 数值错 | 中 |
| 23 | 滴定体积不在 20-40 mL 之间 | 相对读数误差大 | 中 |
| 24 | 平行测定极差 > 0.2% 仍取均值 | 数据不可靠 | 中 |
| 25 | 算 mass 时漏掉稀释倍数或反滴定扣除 | 数值系统偏差 | 中 |

### 1.5 读数视差类（最被低估的）

| # | 错误 | 后果 |
|---|---|---|
| 26 | **滴定前仰视读数** | 读数偏大，V_起 偏大 → V = V_终 - V_起 偏小 → 结果偏低 |
| 27 | **滴定前俯视读数** | 读数偏小，V_起 偏小 → V 偏大 → 结果偏高 |
| 28 | **滴定后仰视读数** | 读数偏大，V_终 偏大 → V 偏大 → 结果偏高 |
| 29 | **滴定后俯视读数** | 读数偏小，V_终 偏小 → V 偏小 → 结果偏低 |
| 30 | 读数时未等 30 秒 / 1 分钟（管壁挂液） | V 读数虚增 |

**记忆口诀**（教学可用）：
- 滴定管"0 在上"——**仰视读数大，俯视读数小**
- 量筒"0 在下"——**仰视读数小，俯视读数大**（与滴定管相反）

## 2. 模拟器已建模的 vs 待建模

| 错误 | 已建模 | 待建模（建议优先级） |
|---|---|---|
| 缓冲液失效 | ✅ Stage 2 完整模拟 | — |
| 用错数据源（理论 vs 标定） | ✅ Stage 2 校验逻辑 | — |
| 过量滴定 | ✅ Stage 1 阈值判定 | — |
| 滴定管未润洗 | ❌ | 🟡 P2（中量，反思页文案） |
| 锥瓶误润洗 | ❌ | 🟡 P2（反思页文案） |
| 半滴控制不规范 | ✅ Stage 1 滑块步长在末端自动改 0.5 → 但 0.05 步长按钮已存在 | 🟢 P3（轻量，UI 引导） |

| 流速过快 | ⚠️ 部分（Stage 1 滑块步长 0.1 mL 在近端变 0.5） | 🟢 P3 |

| 平行测定极差 | ❌ | 不适合单次模拟器 |

## 3. 落地方案：A + B（推荐）

### 3.1 A 方案 — 操作规范 onboarding 卡片

**位置**：在 Stage 1 入口（OnboardingBanner 之上或之内）

**形式**：可折叠的卡片，4 个图标 + 一句话说明

**文案草稿**：

```
┌──────────────────────────────────────────┐
│ 标准滴定操作规范 · 4 件事                │
│ ──────────────────────────────────────   │
│ 🧪 滴定管润洗                            │
│    用标准液润洗 3 次。残留水会稀释      │
│    浓度，最终结果偏高                    │
│                                          │
│ 👀 平视读数                              │
│    视线与弯月面最低点相切。              │
│    仰视/俯视都会引入 ±0.01 mL 误差     │
│                                          │
│ 💧 半滴控制                              │
│    近终点时让液滴悬在尖嘴，              │
│    用锥瓶内壁沾下，纯水冲入瓶中         │
│                                          │
│ 🌀 摇瓶                                  │
│    圆周摇动，让反应均匀。                │
│    摇不够 = 反应不完全                   │
└──────────────────────────────────────────┘
```

**实现要点**：
- 在 `OnboardingBanner.tsx` 里加一张"操作规范"卡片，仅在 Stage 1 显示
- 默认折叠，点击展开
- 不要塞进 Stage 2（Stage 2 重点是"系统监控"和"数据继承"，再加规范就过载了）

### 3.2 B 方案 — 通关反思强化

**位置**：`ResultModal.tsx` 中"你做对了什么"之下，加一个"避坑清单"区

**文案草稿**：

```
┌──────────────────────────────────────────┐
│ 🛡 如果你刚才犯了这些错 ·                │
│    WHAT MISTAKES YOU AVOIDED              │
│ ──────────────────────────────────────   │
│                                          │
│ ❌ 滴定管未用标准液润洗                  │
│    → V 偏大，结果偏高 ~0.5-1%           │
│                                          │
│ ❌ 锥形瓶误用待测液润洗                  │
│    → 实际样品量翻倍，结果偏高 ~100%     │
│                                          │
│ ❌ 近终点没改用半滴                      │
│    → 过量 0.05 mL，结果偏高             │
│                                          │
│ ❌ 仰视/俯视读数                         │
│    → ±0.01 mL 系统偏差                   │
│                                          │
│ ❌ 忘了加缓冲液                          │
│    → pH 崩溃，指示剂锁死在粉色         │
│                                          │
│ ❌ Fe³⁺ / Al³⁺ 未掩蔽                   │
│    → 铬黑 T 被"封闭"，永远不变蓝       │
│                                          │
│ ❌ 套用理论浓度 0.05 而非标定值          │
│    → 数据源错误，结果完全错             │
└──────────────────────────────────────────┘
```

**实现要点**：
- 在 `ResultModal.tsx` 当前 "你做对了什么" 区块后追加一个新区块
- 用 `AlertTriangle` 或 `ShieldAlert` 图标区分
- 文案简短（每条 ≤ 25 字）
- **不要做交互**——这是反思页，让用户读完就够

### 3.3 为什么不做 Stage 3（C 方案）

考虑过的 Stage 3 设计："操作挑战赛"——给定一个目标浓度，让玩家避开多种陷阱完成滴定。

**不做原因**：
1. **复杂度爆炸**：要同时建模 5+ 种错误及其组合效应，单 session 难完成
2. **教学价值稀释**：把"操作规范"做成关卡，反而变成"记忆陷阱位置"的找茬游戏
3. **真实错误常常组合发生**：模拟器简化后可能失真，反而误导
4. **A+B 已经覆盖核心**：每条错误都在反思页给出"原因 + 后果"，足够朋友记住

**保留可能性**：如果未来用户规模扩大、或作者想做更复杂版本，可作扩展。详见 §6。

## 6. 未来扩展（待作者决策）

### 6.1 Stage 3 "操作挑战赛"（未做）

**设想**：
- 玩家拿到一个未知 Zn²⁺ 样品
- 在限定操作步数内完成精确滴定
- 每步操作都有"陷阱版"和"正确版"两套参数
- 评估总分（精度 + 避错数）

**好处**：
- 把"操作规范"从"被动阅读"变成"主动练习"
- 覆盖 Fe³⁺ 封闭、视差、润洗等难演示的错误

**坏处**：
- 复杂度高（要做操作评分系统）
- 单 session 内难完成

**作者决策**：要不要做 Stage 3？

### 6.2 Fe³⁺ 封闭的专门 Stage

**设想**：新增 Stage 3 = "污染排查"。场景：水样里含有微量 Fe³⁺，EBT 直接失效。玩家必须加入三乙醇胺掩蔽才能继续。

**好处**：
- 这是真实实验里**最容易翻车**的细节之一
- 把"掩蔽剂"这个概念直观呈现

**实施成本**：中（要改 store 加 maskant toggle，要加新 pH 阈值判定）

### 6.3 把视差做成可视化

**设想**：在 PHChart 或 TelemetryCards 上方加一个"读数视差演示器"——展示俯视 / 平视 / 仰视三种读数差异。

**好处**：最被低估的错误之一，可视化冲击力强

**实施成本**：低（静态 SVG 即可）
