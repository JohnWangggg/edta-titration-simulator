## 项目名称：滴定系统遥测与失效模拟器 (Titration System Failure Simulator)

**Author:** 楼楼大王
**项目目标：** 开发一个单页 Web 应用（SPA），通过交互式滑块和实时折线图，向毫无化学基础的用户直观展示“缓冲溶液失效”是如何导致 EDTA 络合滴定实验彻底崩溃（指示剂颜色物理锁死）的。

## 1. 技术栈建议

* **框架：** React (Next.js App Router 尤佳，方便快速部署)
* **样式：** Tailwind CSS (构建工业风/遥测仪表盘风格 UI)
* **图表：** Recharts (用于绘制 pH 值随滴定体积变化的动态折线图)
* **状态管理：** React Hooks (`useState`, `useEffect`)

## 2. 核心状态与底层逻辑 (State & Logic)

需要维护以下核心全局状态：

* `volume` (float): 当前滴入的 EDTA 体积，范围 0.00 ~ 30.00 mL，初始 0.00。
* `bufferEnabled` (boolean): 缓冲溶液开关，初始 `false`。
* `pH` (float): 体系的实时 pH 值。
* `color` (string): 锥形瓶内溶液的实时颜色 HEX 值。

**【核心物理引擎 / 状态计算函数】**
每当 `volume` 或 `bufferEnabled` 改变时，重新计算 `pH` 和 `color`：

**逻辑分支 A：缓冲溶液【开启】 (Ideal State)**

* **pH 计算：** 始终死死锁定在 `10.0`，无论 `volume` 怎么变。
* **颜色计算：**
* `volume < 19.5` -> 酒红色 (Wine Red)
* `19.5 <= volume < 20.0` -> 紫红色过渡 (Purple)
* `volume >= 20.0` (理论终点) -> 纯蓝色 (Clear Blue)



**逻辑分支 B：缓冲溶液【关闭】 (System Failure State - 重点重现)**

* **pH 计算：** EDTA 释放 H+，导致 pH 骤降。模拟公式可设为对数衰减：
* 当 `volume == 0`，`pH = 7.0`
* 当 `volume > 0`，`pH = Math.max(3.5, 7.0 - Math.log10(volume * 10 + 1) * 2)` (确保 pH 会迅速掉到 4 左右)


* **颜色计算 (The Bug)：** 铬黑T指示剂的致命弱点。
* 如果 `pH < 6.5` -> 强制锁定为亮粉红色 (Hot Pink)，无视 `volume` 的值。即使 `volume` 超过 20.0，也永远保持粉红（模拟越滴越粉的死锁）。
* 如果 `pH >= 6.5` 且 `volume < 20.0` -> 酒红色。



## 3. UI/UX 组件布局 (Component Structure)

界面采用深色模式（Dark Mode），营造“高精度生物遥测仪器”的质感，分为左中右三个区块：

**左侧：控制台 (Control Panel)**

* **主控开关：** 一个醒目的 Toggle Switch，“注入氨-氯化铵缓冲溶液 (锁定 pH=10)”。
* **滴定执行器：** 一个 Range Slider，控制 `volume` (0~30mL)，步长 0.1。旁边配有一个“+1滴 (0.05mL)”的微调按钮。
* **重置按钮：** “系统归零 (Reset)”。

**中间：视觉反馈区 (Visual Feedback)**

* **烧瓶视窗：** 一个极简的 SVG 锥形瓶图形。瓶内液体的填充颜色 (`fill`) 绑定全局 `color` 状态。
* **大字号状态文本：** 实时显示当前颜色状态（如：“未到达终点”、“系统已死锁”、“滴定成功”）。

**右侧：遥测数据流 (Telemetry Dashboard)**

* **核心指标卡：** 两个大型数字面板，分别显示 `实时 pH 值`（保留两位小数）和 `已消耗 EDTA (mL)`。
* **pH 跌落监控图：** 使用 Recharts 绘制的实时折线图。
* X 轴：EDTA 体积。
* Y 轴：pH 值 (0 - 14)。
* 动态绘制当前滴定轨迹。添加一条醒目的横向警戒线在 `pH = 6.5`，标注“指示剂失效红线”。



## 4. 交互教学引导 (Onboarding)

在 UI 顶部增加一个帮助提示框，引导用户进行两轮测试：

1. **第一轮：** 不要开缓冲溶液，直接拉动滑块，观察系统是如何因为 pH 值跌破红线而崩溃锁死在粉红色的。
2. **第二轮：** 点击重置，开启缓冲溶液，仔细观察在 20mL 处的完美颜色突变。

**提示给开发者的系统工程思考：**
在实现“越滴越粉”的逻辑时，重点向 AI 强调 `pH < 6.5` 这个条件拥有**最高优先级的颜色重写权**（Override）。这正是现实化学实验中发生的事情：底层环境变量的崩溃，直接导致了上层表象逻辑的失效。


坦白说，**并没有完全覆盖**。

刚才给你的那份需求文档（PRD）属于“单点突围”——它像一把尖刀，专门针对了你们在【实验二】中最致命的那个物理 Bug（没加缓冲液导致 pH 崩溃、指示剂锁死）。

但如果你回看你们真实的实验记录本，你们其实犯了**两个独立且致命的错误**：

1. **化学系统错误（实验二的越滴越粉）：** 刚才的 PRD 已经完美覆盖了。
2. **逻辑与精度错误（实验一的计算脱节）：** 你们在实验一算出了实际浓度（0.0593），但在实验二里却套用了理论浓度（0.05）。刚才的系统里并没有强制要求用户去“使用计算结果”。

## 补充需求：双关卡闯关模式 (Two-Stage Progression System)

**修改目标：** 将单页面应用升级为“实验一（标定） -> 实验二（测定）”的连续逻辑闭环。强制用户体验“精度控制”以及“数据继承”的重要性。

### 新增全局状态：

* `currentStage` (1 或是 2): 当前处于实验一还是实验二。
* `actualEdtaConcentration` (float): 系统在初始化时**暗中随机生成**的一个真实浓度（范围 0.0450 ~ 0.0650，例如 0.0593）。这个值对用户是隐藏的。

### 关卡 1：实验一 (0.05mol/L EDTA 标定)

**核心训练点：半滴精度控制与浓度计算**

* **场景设定：** 锥形瓶内是 25.00 mL 的已知浓度锌标准液 (0.0500 mol/L)。默认已加缓冲液（无需用户操作）。
* **交互要求：** * 用户拖动滑块滴加 EDTA。当接近理论终点时，如果用户拖动滑块太快（步长过大），直接判定为“滴定过量（深蓝色）”，要求重做。
* 强制用户在最后阶段使用“+0.05mL (半滴)”按钮微调。


* **通关条件：** 滴定至准确的纯蓝色后，弹出一个输入框：“请根据消耗体积，计算并输入 EDTA 的真实浓度（保留四位小数）”。
* **校验逻辑：** 用户输入的值必须与系统暗中生成的 `actualEdtaConcentration` 匹配（允许 $\pm 0.0002$ 误差）。输入正确后，解锁并进入关卡 2。

### 关卡 2：实验二 (ZnSO₄ 试样中 Zn²⁺ 含量测定)

**核心训练点：系统遥测监控（防崩溃）与数据继承**

> **2026-06-18 修订**：原 PRD 写的是"硫酸钠试样"，但 EDTA 不能直接滴定 SO₄²⁻（EDTA 只与金属阳离子螯合，SO₄²⁻ 是阴离子）。真实方法是用 BaCl₂ 沉淀 SO₄²⁻ 后再用 EDTA 反滴过量 Ba²⁺（SL 85-1994 国标），过于复杂不适合教学。
>
> **改用 ZnSO₄ 试样**：EDTA 直接滴定 Zn²⁺（pH 10，铬黑 T 指示剂），化学无瑕疵。同时 Stage 1 标定的就是 Zn²⁺，数据继承的训练逻辑天然成立。

* **场景设定：** 测定 0.5000 g 含 ZnSO₄ 的未知固体试样（含有不参与反应的情性杂质）。界面出现我们在上一版设计的【氨-氯化铵缓冲溶液】开关。
* **交互要求 (复刻之前的逻辑)：** * 如果不打开开关，滴定必定进入"越滴越粉"的死锁状态，UI 提示"系统 pH 崩溃，请排查原因"。
* 必须先打开开关，再进行滴定，才能得到纯蓝色终点。

* **计算说明**（化学原理）：
* 25.00 mL 锥瓶里装的是溶解后的 ZnSO₄ 试样溶液
* Zn²⁺ 与 EDTA 按 1:1 络合：n(Zn²⁺) = c(EDTA) × V(EDTA)
* m(ZnSO₄) = n(Zn²⁺) × M(ZnSO₄) = c(EDTA) × V(EDTA) × 161.47 g/mol
* ZnSO₄ 摩尔质量 = 161.47 g/mol（Zn: 65.38, S: 32.07, O₄: 64.00）
* 质量分数 = m(ZnSO₄) / 0.5000 × 100%

* **终极通关条件 (填补真实实验的逻辑漏洞)：**
* 滴定成功后，弹出最终的质量分数计算框。
* **关键陷阱：** 系统故意在旁边放一个提示语"已知 EDTA 理论浓度为 0.05 mol/L"。
* **校验逻辑：** 如果用户输入了 0.05，提示"计算错误：你忘记了关卡 1 中辛苦标定出的真实浓度了吗？"。只有当用户代入**关卡 1 计算出的真实浓度**（系统校验的是 c(EDTA) 这个数值，误差 ±0.0005 mol/L），才能得出正确结果，达成通关。
